Product Explainer

How this demo works

Predicts whether a county will be in the highest-risk 25% for eviction filing rate next year. Historical demo using county data with outcomes through 2018.

See the analysis (charts + backtests)

The question we answer

  • Input: recent county filing-rate history and simple trend summaries.
  • Output: the chance a county lands in next year’s top-risk quartile.
  • Percentile interpretation: 92nd percentile means higher risk than about 92% of counties in that year.
  • If you leave year blank, the API uses the latest available year for that county.

How it works

1

Build features from recent history

We summarize recent filing-rate behavior into a few stable county-level inputs.

2

Weighted linear score (log-odds)

The model combines those inputs with learned weights to create a single risk signal.

3

Sigmoid converts to probability (0–1)

That signal is mapped to a probability-like score that is easy to compare across counties.

What we learned

Eviction filing rates appear persistent from year to year. In backtests, the full model performs strongly, while a naive baseline using only lag_1 is still close. That implies recent history carries a lot of signal, and the richer feature set adds incremental value.

What goes into the model

lag_1

Most recent observed filing rate for the county.

lag_3_mean_obs

Average filing rate across the three most recent observed years.

lag_5_mean_obs

Average filing rate across five observed years for a longer trend view.

years_since_last_obs

How long since the county’s previous observation in the irregular panel.

Limitations and responsible use

  • Outcomes in this dataset run through 2018.
  • Predictions are county-level only, not neighborhood or household level.
  • This is not a causal model and should not be read as cause-and-effect.
  • Demo only. Do not use for high-stakes decisions.
Technical details

Logistic regression models the log-odds of the outcome as a weighted sum of features, then converts that value to a probability with the sigmoid function.

Readable formula: logit = b0 + b1*lag_1 + b2*lag_3_mean_obs + b3*lag_5_mean_obs + b4*years_since_last_obs, then p = 1 / (1 + exp(-logit)).

We use regularization (C=0.1) to avoid unstable weights and calibration guardrails to avoid overconfident probability saturation.
